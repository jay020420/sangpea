# PDP Generation Failure Report

작성일: 2026-06-24

## 요약

첨부 결과물은 상세페이지 섹션으로 사용할 수 없다. 문제는 단순히 상세페이지 레퍼런스가 부족해서가 아니라, 생성 단계가 시각 에셋과 편집 가능한 카피/도형/CTA 레이어의 책임을 충분히 분리하지 못한 구조에서 발생했다.

가장 큰 실패는 모델이 최종 카피, CTA, 제품명처럼 수정되어야 할 정보를 이미지 안에 직접 그렸다는 점이다. 이렇게 생성된 텍스트는 오탈자, 의미 왜곡, 가독성 저하, 수정 불가능 문제를 동시에 만든다.

## 관찰한 실패 증상

- 생성 이미지 안에 `KillKagi`, 한국어 헤드라인, CTA 버튼 문구, 말풍선형 문구가 직접 렌더링됐다.
- 상세페이지 섹션이라기보다 SaaS 대시보드 포스터 또는 앱 광고 시안처럼 보인다.
- 제품 또는 서비스의 실제 화면/기능 구조가 명확하지 않고, 가짜 대시보드 수치와 UI 패널이 많다.
- 텍스트가 이미지 픽셀로 굳어 있어 편집기에서 문구만 수정하는 실무 흐름에 맞지 않는다.
- 모바일 PDP에서 중요한 정보 위계인 제품/화면 노출, 편집 가능한 카피 레이어, CTA 가시성이 모두 불안정하다.

## 원인

1. 생성 프롬프트의 출력 계약이 약했다.
   - 기존 프롬프트에도 "앱이 카피를 합성한다"는 설명은 있었지만, 모델 입장에서는 완성 상세페이지처럼 보이게 텍스트를 직접 그릴 여지가 남아 있었다.

2. 시각 에셋과 최종 문서 레이어의 책임이 섞였다.
   - 이미지 생성 모델은 제품, 화면, 사용 장면, 증빙 컷 같은 시각 에셋만 만들어야 한다.
   - 배경 도형, 카드, 헤드라인, 불릿, CTA, 증빙 문구는 React 편집기 레이어가 담당해야 한다.

3. 품질 게이트가 텍스트 베이킹과 가짜 UI를 충분히 강제 차단하지 못했다.
   - 점수형 PDP 체크가 낮아도 구조화 이슈가 약하면 재생성 기준이 느슨해질 수 있었다.

4. SW/SaaS 입력에서 가짜 대시보드 생성 위험이 높았다.
   - 실제 스크린샷이 충분하지 않거나 프롬프트가 강하지 않으면 모델은 그럴듯한 가짜 대시보드를 만든다.

5. UI도 문제를 악화했다.
   - 첫 화면 안내가 길고 패널이 많아 사용자가 어떤 자료를 넣어야 품질이 좋아지는지 빠르게 판단하기 어려웠다.

## 적용한 수정

- 이미지 생성 프롬프트를 `BACKGROUND PLATE`가 아니라 `VISUAL ASSET` 계약으로 전환했다.
- 새 한국어/영어 문구, 앱 이름, CTA, 말풍선, 배지, 대시보드 숫자, 가짜 UI 텍스트를 절대 생성하지 못하도록 명시했다.
- 텍스트 영역, 카드, CTA, 증빙 표면은 생성 이미지가 아니라 편집 가능한 문서 레이어가 담당하도록 변경했다.
- SW/SaaS에서 실제 스크린샷이 없으면 상세 가짜 대시보드를 만들지 않도록 제한했다.
- 품질 게이트에서 layerEditability, textReadability, productExposure, whitespaceBalance 점수가 낮으면 자동 재생성되도록 강화했다.
- 점수형 품질 체크가 낮을 때 서버가 직접 major/critical 이슈를 합성하도록 보강했다.
- RAG 시드 문서를 "텍스트 없는 배경 플레이트"가 아니라 "문서 레이어와 분리된 시각 에셋" 원칙으로 정리한다.
- 업로드 UI를 3단계 Workflow 구조로 바꾸고, 장문 안내를 줄였다.

## 남은 리스크

- 이미지 모델은 여전히 일부 글자 형태를 만들 수 있다. 따라서 최종적으로는 OCR 기반 텍스트 검출이나 비전 모델 2차 판정이 필요하다.
- 실제 제품/서비스 화면 레퍼런스가 부족하면 "좋은 시각 에셋"은 만들 수 있어도 "정확한 상품 상세페이지"는 만들기 어렵다.
- ComfyUI/segmentation을 붙이기 전까지 제품 컷과 배경의 완전한 계층 분리는 제한적이다.

## 다음 단계

- 생성 결과에 OCR 또는 비전 기반 "새 텍스트 검출" 게이트를 추가한다.
- 차단된 이미지가 편집기에 들어오면 바로 재생성 사유와 수정 프롬프트를 보여준다.
- 제품/화면/사용 장면 시각 에셋과 카피/카드/CTA 노드를 분리하는 파이프라인을 강화한다.
- 실제 경쟁사 PDP, 후기, 카테고리별 구조 예시를 Knowledge로 계속 보강한다.
