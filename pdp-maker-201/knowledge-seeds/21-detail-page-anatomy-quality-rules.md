제품 사실 대체 금지: 이 문서는 상세페이지의 구조와 품질 기준만 제공한다. 실제 제품명, 기능, 효능, 수치, 인증, 후기, 가격, 배송 조건, 고객사, 보안 인증은 업로드 자료나 사용자가 제공한 정보에서 확인될 때만 쓴다.

목적: "상세페이지스럽다"는 의미를 생성 모델이 이해하도록 강제한다. 상세페이지는 예쁜 한 장짜리 포스터가 아니라 모바일에서 스크롤하며 구매 또는 도입 결정을 돕는 설명 흐름이다.

## 상세페이지의 정의

상세페이지는 구매자가 첫 화면에서 제품 정체와 선택 이유를 이해하고, 이어지는 섹션에서 의심을 줄이며, 마지막에 행동할 수 있게 만드는 세로형 판매 설명서다.

좋은 상세페이지 섹션은 광고 포스터처럼 분위기만 보여주지 않는다. 제품 또는 서비스가 무엇인지, 누구에게 필요한지, 어떤 상황에서 쓰는지, 왜 믿을 수 있는지, 구매 전 무엇을 확인해야 하는지를 화면 안의 정보 위계로 보여준다.

한국 이커머스 상세페이지는 모바일 가독성이 중요하다. 한 섹션에는 메시지 하나만 담고, 헤드라인은 짧게, 보조 카피는 1~2문장으로 제한하며, 긴 설명은 여러 섹션으로 나눈다.

## 섹션 이미지 anatomy

Hero 섹션: 제품 또는 서비스 화면이 가장 먼저 보인다. 한 줄 헤드라인, 짧은 보조 카피, 핵심 신뢰 단서, CTA 영역이 있어야 한다. 단순 감성 배경이나 모델 사진만 있으면 실패다.

Problem 섹션: 고객이 겪는 상황을 구체적으로 보여준다. 체크리스트, 질문 카드, 사용 전 불편 장면을 사용한다. 공포를 과장하거나 근거 없는 문제를 만들지 않는다.

Benefit 섹션: 기능명을 그대로 나열하지 않고 고객 이득으로 번역한다. 3개 이하 카드, 짧은 제목, 한 줄 설명, 관련 제품/화면 디테일을 배치한다.

Proof 섹션: 증빙 이미지, 스펙, 구성품, 성분, 포트, UI 화면 일부처럼 확인 가능한 근거를 보여준다. 인증마크, 별점, 테스트 수치, 후기 말풍선은 자료가 있을 때만 쓴다.

Usage/Demo 섹션: 제품은 사용 장면, SW는 실제 화면 기반 2~4단계 흐름을 보여준다. 구매자가 "내가 쓰면 이렇게 되겠구나"라고 상상할 수 있어야 한다.

Spec/FAQ 섹션: 사이즈, 구성, 주의사항, 배송/교환/AS 또는 보안/요금제/연동/지원 범위를 정리한다. 제공 자료가 없으면 확정 표현 대신 확인 필요로 남긴다.

## 카피 품질 기준

한국어 카피는 짧고 납득 가능해야 한다. "역대급", "완벽한", "무조건", "100%", "1위", "인생템" 같은 표현은 자료가 없으면 쓰지 않는다.

좋은 헤드라인은 제품군, 사용 맥락, 얻는 결과 중 최소 하나를 포함한다. 예: "주문 문의를 한 화면에서 정리", "매일 쓰는 케이블을 더 쉽게 보관", "처음 도입해도 따라가는 3단계 설정".

좋은 보조 카피는 감탄보다 이유를 설명한다. 예: "구성품과 사용 순서를 한눈에 확인할 수 있게 정리했습니다." "보안과 연동 범위는 도입 전 확인할 수 있습니다."

이미지 안에 긴 문장을 직접 굽지 않는다. 핵심 헤드라인과 라벨만 이미지 안에 넣고, 수정이 필요한 가격, 수치, 주의사항, CTA는 편집기 텍스트 레이어로 두는 편이 안전하다.

## 제품 이해 기준

여러 장의 참조 이미지를 받으면 대표컷만 보지 않는다. 디테일컷은 형태와 스펙, 증빙컷은 근거, 기존 상세페이지/PDF는 원본 카피와 판매 조건, SW 스크린샷은 실제 기능과 화면 구조를 이해하는 데 쓴다.

모델은 제품 또는 서비스 이해보다 우선하지 않는다. SW, SaaS, 앱, B2B 서비스는 사람 모델이 필요하지 않을 수 있으며, 실제 화면, 브라우저 프레임, 모바일 프레임, 업무 흐름 카드가 더 적합하다.

제품이 무엇인지 불명확하면 카테고리 주장보다 "제공 자료 기준 확인 가능한 형태와 사용 맥락"을 중심으로 설계한다. 모르는 브랜드명, 기능명, 인증, 가격을 만들어내지 않는다.

## 이미지 생성 실패 패턴

상세페이지스럽지 않은 결과: 분위기 좋은 배경에 큰 제목만 있는 포스터, 제품이 작게 묻힌 광고 이미지, 제품 이해 없이 랜덤한 모델을 배치한 이미지, 반복되는 동일 레이아웃, 읽을 수 없는 긴 문단, 가짜 인증/리뷰/수치를 넣은 이미지.

좋은 결과: 제품 또는 화면이 선명하고, 섹션 목적이 다르며, 텍스트 영역이 읽기 쉽고, 구매자가 다음 정보를 확인할 수 있고, 이어 붙였을 때 하나의 상세페이지 흐름처럼 보인다.
