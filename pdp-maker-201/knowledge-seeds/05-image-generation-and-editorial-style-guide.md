목적: PDP 이미지 생성과 섹션 리디자인에서 일관된 품질, 가독성, 제품 정확성을 유지하기 위한 시각/카피 가이드다. 제품 사실 대체 금지: 이 문서는 제품에 없는 기능, 인증, 수치, 후기, 구성품을 만들어내는 근거가 아니다.

## 이미지 생성 기본 원칙

제품이 주인공이어야 한다. 배경, 모델, 소품, 타이포그래피는 제품의 형태와 구매 이유를 돕는 보조 요소다.

생성 이미지에서 제품의 핵심 형태, 색상, 패키지, 구성품을 임의로 바꾸지 않는다. 특히 로고, 라벨, 성분표, 인증마크, 규격 표시는 원본 자료가 없으면 새로 만들지 않는다.

상세페이지 섹션 이미지는 한눈에 읽혀야 한다. 모바일 기준으로 텍스트는 짧고 굵게, 보조 설명은 1~2문장 이하로 제한한다.

## 참조 이미지 사용 규칙

참조 제품 이미지는 제품 사실의 1순위다. 모델은 제품 형태, 색상, 패키지, 로고 위치, 구성품 수량, 크기 비율을 유지해야 한다.

참조 이미지가 흐리거나 제품 일부만 보이면 새 사실을 추측하지 않는다. 보이지 않는 뒷면, 내부 구조, 성분표, 포트, 구성품은 "제공 자료 기준 확인 불가"로 남긴다.

모델 컷이나 사용 장면을 만들 때도 제품은 같은 제품처럼 보여야 한다. 제품 색상 변경, 용기 형태 변경, 패키지 재디자인, 임의 라벨 생성은 실패로 본다.

소프트웨어, 앱, 웹서비스, SaaS 화면이 참조로 제공되면 스크린샷이 제품 사실의 1순위다. UI 구조, 메뉴명, 화면 색상, 실제 텍스트, 차트 형태, 버튼 위치를 보존하고, 제공되지 않은 기능 화면, 가짜 고객 데이터, 가짜 로고, 가짜 보안 배지, 가짜 매출 그래프를 만들지 않는다.

## 섹션별 구도 기준

Hero: 제품 클로즈업, 깨끗한 배경, 핵심 헤드라인 1개, 짧은 보조 카피 1개. 제품은 화면의 40~70%를 차지한다. 첫 화면에서 제품군과 구매 이유가 보여야 한다.

Benefit: 장점 하나당 하나의 장면. 장점은 고객 이득과 연결한다. 아이콘 또는 짧은 라벨은 가능하지만 인증처럼 보이는 배지는 실제 근거가 있을 때만 쓴다.

Proof: 소재, 구조, 원재료, 사용법, 구성품, 디테일을 확대 컷으로 보여준다. 테스트, 인증, 임상, 비교 수치는 제공 자료가 있을 때만 시각화한다.

Usage: 실제 사용 환경을 보여준다. 너무 연출된 광고 사진보다 구매자가 자기 상황을 떠올릴 수 있는 자연스러운 장면이 좋다. 사용자가 제공하지 않은 인물, 장소, 계절 맥락은 제품군에 맞는 일반 장면으로만 제한한다.

Spec: 표, 리스트, 구성품 나열은 깔끔하게 정리한다. 작은 글씨가 너무 많으면 섹션을 나눈다. 사이즈, 용량, 소재, 호환성은 원본 자료가 있을 때만 확정한다.

FAQ: 질문과 답변을 분리하고, 구매 불안을 해결하는 짧은 문장으로 구성한다. 배송, 교환, AS, 사용 전 주의사항은 판매 조건이 확인될 때만 확정한다.

Software/SaaS: 제품 화면, 앱 스크린샷, 웹 대시보드, 브라우저 프레임, 모바일 프레임, 업무 흐름 카드, Before/After 워크플로우, 기능 콜아웃을 중심으로 구성한다. 가격, 요금제명, 보안 문구, 기능명, CTA는 가능하면 편집기 텍스트 레이어로 둔다.

## 텍스트 오버레이 규칙

이 앱의 섹션 이미지 생성은 최종 완성본이 아니라 배경 플레이트를 만드는 단계다. 헤드라인, 서브카피, 불릿, CTA, 가격, 후기, 인증, 숫자 근거는 편집기 레이어가 합성한다.

생성 모델은 새 텍스트를 이미지 안에 굽지 않는다. 새 한국어/영어 문장, 앱 이름, CTA 단어, 말풍선, 배지 라벨, 대시보드 숫자, 리뷰 문구가 픽셀로 생성되면 실패다. 단, 업로드된 제품 패키지나 실제 SW 스크린샷 안에 원래 있던 텍스트는 원본 일부로 유지할 수 있다.

텍스트가 들어갈 곳은 빈 카드, 빈 패널, 여백, 추상 막대, 중립 아이콘, 버튼 모양의 빈 영역으로 남긴다. 최종 카피가 들어갈 안전영역은 모바일 기준으로 충분한 대비와 패딩을 가져야 한다.

중요한 한국어 텍스트는 이미지 안에 굽지 말고 편집기 오버레이로 둔다. 가격, 용량, 인증, 구성, CTA, 배송, 교환, 주의사항, 헤드라인은 수정 가능한 텍스트 레이어가 우선이다.

텍스트는 제품의 중요한 부분을 가리지 않는다. 제품 위에 직접 글자를 얹어야 한다면 반투명 배경, 충분한 여백, 높은 대비를 사용한다.

한 줄 제목은 8~18자 정도가 가장 안정적이다. 긴 문장은 이미지 안이 아니라 편집기의 별도 텍스트 블록으로 배치한다.

글자가 배경과 겹치거나 깨지면 이미지 전체 재생성보다 텍스트 위치, 색상, 줄바꿈, 배경 사각형을 먼저 조정한다.

## 스타일 방향

프리미엄: 낮은 채도의 배경, 큰 여백, 소재 클로즈업, 절제된 타이포그래피, 고급 조명. 저렴한 장식보다 제품 마감과 사용 경험을 보여준다.

실용/가성비: 제품 구성, 사용 단계, 전후 정리감, 가격 대비 구성, 명확한 표. 가격 표현은 실제 판매 조건이 있을 때만 쓴다.

생활감/감성: 실제 공간, 손이 닿는 장면, 계절감, 자연광, 따뜻한 톤. 단, 제품 확인성이 떨어지면 안 된다.

테크/디지털: 차분한 그레이 또는 어두운 배경, 선명한 하이라이트, 스펙 카드, 호환 기기 레이아웃, 포트/버튼/규격 클로즈업.

소프트웨어/SaaS: 실제 스크린샷을 브라우저 또는 모바일 프레임 안에 배치하고, 화면 옆에는 기능 콜아웃과 3단계 워크플로우를 둔다. 대시보드 숫자, 고객사 로고, 보안 인증, 연동 서비스 로고는 제공된 자료가 있을 때만 시각화한다.

키즈/패밀리: 밝고 안전한 톤, 부드러운 색, 부모가 확인해야 할 정보 강조. 과장된 안전 표현과 무근거 인증은 피한다.

## 금지 스타일

제품보다 장식이 더 큰 이미지.
무근거 인증마크, 임상 배지, 리뷰 별점, 공식 로고 생성.
읽기 어려운 긴 문단을 이미지 안에 삽입.
과도한 네온, 복잡한 배경, 제품 색상 왜곡.
의료/건강 효과를 시각적으로 암시하는 전후 비교.
실제 상품에 없는 구성품, 색상, 패키지, 모델, 사용 장면 추가.
영상 광고처럼 과도한 감탄사, 극단적 후킹, 허위 긴급성 표현.

## 프롬프트 작성 규칙

프롬프트에는 섹션 목적, 제품 유지 조건, 참조 이미지 사용 방식, 시각 스타일, 텍스트 처리, 금지 사항을 함께 넣는다.

Hero 예시:
"참조 이미지의 제품 형태와 색상을 유지한다. 모바일 상세페이지 Hero 섹션. 제품을 화면 중앙에 크게 배치하고 깨끗한 밝은 배경을 사용한다. 헤드라인은 편집기 오버레이로 처리한다. 인증마크, 리뷰 별점, 할인율, 실제와 다른 구성품을 만들지 않는다."

Benefit 예시:
"참조 제품을 유지하고 사용자가 체감할 장점 하나만 시각화한다. 생활 장면은 자연스럽게, 텍스트는 짧은 라벨 수준으로 제한한다. 제품에 없는 기능이나 효과를 암시하지 않는다."

Proof 예시:
"제품의 소재, 구조, 구성품 중 원본 자료에서 확인되는 요소만 클로즈업한다. 임상, 인증, 1위, 테스트 완료 배지는 만들지 않는다. 제공 자료 기준 확인 가능한 디테일만 보여준다."

Spec/FAQ 예시:
"모바일에서 읽기 쉬운 표 형태의 섹션 배경을 만든다. 실제 수치와 문구는 편집기 텍스트 레이어로 넣는다. 작은 글씨를 이미지 안에 직접 생성하지 않는다."

Software/SaaS Hero 예시:
"제공된 앱 스크린샷의 UI 구조와 색상을 보존한다. 브라우저 프레임 안에 실제 화면을 배치한 한국형 SW 홍보 상세페이지 Hero 섹션. 헤드라인과 CTA는 편집기 텍스트 오버레이 영역으로 남긴다. 고객사 로고, 보안 인증, 성과 수치, 실제와 다른 기능 화면을 만들지 않는다."

Software/SaaS Demo 예시:
"제공된 웹서비스 화면을 기준으로 3단계 데모 섹션을 만든다. 1 입력, 2 자동 정리, 3 결과 확인의 흐름을 카드로 보여준다. 실제 화면에 없는 버튼명, 요금제, 연동 서비스, 가짜 데이터를 만들지 않는다."

나쁜 프롬프트 예:
"최고의 제품처럼 멋지게 만들어줘. 1위 느낌, 인증 배지, 리뷰 별점, 엄청난 효과 강조."

## 편집기 사용 원칙

생성 이미지에서 중요한 글자가 깨지면 이미지 재생성보다 편집기 텍스트 오버레이로 교체한다.

섹션별 색상과 폰트는 통일하되 모든 섹션이 같은 레이아웃이면 지루하다. Hero, Benefit, Proof, Spec, FAQ의 리듬을 다르게 만든다.

ZIP 다운로드 전에는 각 섹션의 텍스트 겹침, 제품 잘림, 오탈자, 무근거 표현, 제품 색상 왜곡, 실제와 다른 구성품 생성을 확인한다.
