제품 사실 대체 금지: 이 문서는 이미지 연출 방향만 제공한다. 실제 제품 색상, 구성품, 인증, 성능, 기능, 요금제, 사용 조건은 참조 이미지와 제품/SW 자료가 있을 때만 확정한다.

## 기본 방향

제품이 장식보다 커야 한다. 배경, 소품, 효과가 제품을 가리면 실패다.

원본 제품의 형태, 색상, 패키지, 구성품을 유지한다. 없는 색상, 없는 패키지, 없는 라벨, 없는 구성품을 만들지 않는다.

원본이 앱 화면, 웹 대시보드, 프로그램 스크린샷이면 UI 구조와 실제 텍스트를 유지한다. 없는 기능 화면, 가짜 수치, 가짜 고객 데이터, 가짜 보안 배지를 만들지 않는다.

한국 상세페이지 이미지는 모바일에서 빠르게 읽혀야 한다. 텍스트는 짧게, 레이아웃은 여백 있게, 정보 우선순위는 분명하게 만든다.

## 추천 연출

뷰티: 제형, 사용감, 패키지 질감, 세면대/파우치/화장대 같은 현실적인 장면. 의학적 전후 비교는 금지.

식품: 실제 재료, 식탁 장면, 보관/섭취 방법, 패키지 구성. 질병 예방/치료 암시는 금지.

리빙: 사용 전후 공간 정돈, 크기감, 보관 방식, 내구성 근거가 보이는 구조. 과도한 럭셔리 배경은 피한다.

패션: 착용 핏, 소재감, 계절감, 코디 맥락. 실제와 다른 체형 보정이나 색상 왜곡은 피한다.

전자/가전: 포트, 호환, 사용 장면, 크기 비교, 주요 조작부. 공식 인증마크는 원본에 있을 때만 쓴다.

소프트웨어/SaaS: 실제 스크린샷, 브라우저 프레임, 모바일 프레임, 기능 콜아웃, 3단계 데모, 워크플로우 카드. 요금제, 고객사, 보안 인증, 연동 로고는 원본에 있을 때만 쓴다.

키즈/패밀리: 밝고 안전한 톤, 부모가 확인해야 할 정보, 사용 연령/주의사항. 안전 인증은 원본 근거가 있을 때만 쓴다.

## 편집기 연계

이미지 안의 글자가 깨지면 이미지 재생성보다 편집기의 텍스트 오버레이로 교체하는 것이 낫다.

섹션별 색상과 폰트는 통일하되 모든 섹션이 같은 레이아웃이면 지루하다. Hero, Benefit, Proof, Usage, FAQ의 리듬을 다르게 만든다.
